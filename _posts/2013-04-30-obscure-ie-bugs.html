1. 5,000 CSS Rules Per File

Turns out you can't have more than 5,000 style rules in a single CSS file. Hopefully you never run into this, but it can definitely happen when you're bundling library CSS together.

2. 31 Style Tag Limit

Source: http://support.microsoft.com/kb/262161 This one is a lot rarer, but it's definitely something you'll run into if you're working on a more mature codebase. IE7/8 will not allow you to have more than 31 style or link tags on the page. While that's not recommended in production environments, it's not entirely unheard of in dev or QA environments where having everything unbundled is helpful for debugging. 3. Maximum URL Length Certain versions of IE don't allow urls longer than 255 characters. After that, stuff just gets chopped off, resulting in bad paths.
